# 添加角色指南

本文档以「狂战士 (Berserker)」为例，说明如何为本项目添加一个新角色。

---

## 项目架构概述

角色系统采用 **纯数据驱动** 模式——角色不是 class，而是普通的 JS 对象。所有角色放在 `js/characters/` 目录，每个文件导出一个对象。

```
js/characters/
  ├── index.js          ← 注册中心（新增角色必须改这里）
  ├── Swordsman.js
  ├── Berserker.js      ← 示例：狂战士
  └── ...

js/skills/
  ├── SkillRegistry.js  ← 技能注册中心（新增技能类型必须改这里）
  └── abilities/        ← 技能执行器
        ├── AoeMelee.js
        └── Whirlwind.js ← 示例：大风车

js/fighter.js           ← Fighter 类（新增被动效果可能改这里）
js/rendering/FighterRenderer.js ← 渲染器（新增视觉特效改这里）
```

---

## 添加一个角色的完整步骤

### 步骤 1：创建角色数据文件

在 `js/characters/` 下新建一个 `.js` 文件，导出角色对象。

**必填字段：**

```js
export const berserker = {
    id: 'berserker',             // 唯一标识符（蛇形命名）
    name: 'Berserker',           // 英文名
    nameCN: '狂战士',            // 中文名
    color: '#D32F2F',            // 主色（身体颜色）
    secondaryColor: '#FF6F00',   // 副色（描边 / 粒子颜色）
    glowColor: 'rgba(211, 47, 47, 0.55)', // 发光色

    // 数值
    size: 34,                    // 身体半径
    speed: 5.2,                  // 移动速度（px/frame @60fps）
    hp: 100,                     // 生命值（也是最大生命值）
    attackPower: 26,             // 基础攻击力
    attackSpeed: 2.0,            // 基础攻击间隔（秒）
    chargeTime: 0.4,             // 攻击前摇（秒）
    attackRange: 78,             // 攻击范围（px）
    lifesteal: 0.12,             // 吸血比例（0~1）

    // 行为
    movePattern: 'linear',       // 移动模式（见下方选项）
    aiTendency: 'aggressive',    // AI 倾向（aggressive / cautious / balanced）
    weaponType: 'melee',         // 武器类型（melee / ranged）
    projectileType: null,        // 弹幕类型（远程必须指定，近战为 null）

    // 技能
    skill: {
        name: '大风车',          // 技能中文名
        nameEN: 'Whirlwind',     // 技能英文名（可选）
        cooldown: 10,            // 冷却时间（秒）
        damage: 10,              // 技能伤害
        range: 95,               // 技能范围（px）
        type: 'whirlwind',       // 技能类型（见步骤 2）
        duration: 2.0            // 持续时间（秒，硬控 / DoT 等使用）
    },

    // 外观
    drawDecorations: function(ctx, x, y, angle, size, time) {
        // 绘制角色装饰（角、武器、翅膀等）
        // 坐标系：调用时 ctx 原点已平移到 (x, y) 且已旋转 angle
        //         +X 方向 = 角色正前方（朝向敌人）
    }
};
```

**可选字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `isHero: true` | boolean | 标记为英雄角色，UI 中独立分组显示 |
| `hidden: true` | boolean | 不在选择界面显示（用于召唤物） |
| `passives` | array | 被动效果列表，每项 `{id, name, description}` |
| `specialEffects` | array | 特殊机制说明，每项 `{name, description}` |
| `projectileSpeed` | number | 覆盖默认弹幕速度 |

**movePattern 可选值：** `linear`, `keepDistance`, `arc`, `dash`, `zigzag`, `flank`, `wobble`

### 步骤 2：添加技能类型（如果需要新类型）

如果角色的技能类型（`skill.type`）在 `SkillRegistry.js` 中尚不存在，需要：

**2a.** 在 `js/skills/abilities/` 下新建技能执行器：

```js
// js/skills/abilities/Whirlwind.js
import * as EffectLib from '../../effects_lib/index.js';

export function executeWhirlwind(caster, skill, weaponSystem, effectSystem, dx, dy, dist) {
    // caster  = 施法者（Fighter 实例）
    // skill    = 技能数据对象
    // dx, dy, dist = 施法者 → 目标的偏移

    // 1. 播放视觉效果
    EffectLib.addAoeMeleeEffect(effectSystem, caster.x, caster.y, caster.color, skill.range);
    effectSystem.screenShake(10);

    // 2. 对范围内敌人造成伤害
    const enemies = caster.battleContext.opposingTeam;
    if (enemies) {
        enemies.forEach(enemy => {
            if (enemy.isAlive() && /* 在范围内 */) {
                enemy.takeDamage(skill.damage, caster.x, caster.y, effectSystem);
                caster.healFromDamage(skill.damage, effectSystem);
            }
        });
    }
}
```

**可用的 EffectLib 效果函数：**
- `addAoeMeleeEffect(system, x, y, color, radius)` — 扩散环 + 粒子
- `addMeteorEffect(system, x, y, color, radius)` — 陨石坠落
- `addFireBurstEffect(system, x, y, color, radius)` — 火焰爆发
- `addCloneEffect(system, x, y, color, radius)` — 分身烟雾
- `addStunEffect(system, x, y, color, radius)` — 眩晕星星
- `addBackstabEffect(system, x, y, color, radius)` — 背刺闪光

**粒子类型（`effectSystem.addParticle({ type })`）：**
- `'spark'`, `'circle'`, `'ring'`, `'trail'`

#### 技能模式：一次性 vs 持续施法（channeling）

大多数技能是 **一次性** 的（陨石、毒云、突刺），执行器调用一次就结束。但像大风车这种 **持续施法**（引导型技能），项目提供了通用的 `channeling` 状态——不需要为每个技能新增 case。

**2b-1.** 在技能执行器文件中，除了 `executeXxx`（初始爆发），还需导出 `tick` 和 `damage` 两个函数：

```js
// js/skills/abilities/Whirlwind.js

// 初始爆发（技能释放瞬间）
export function executeWhirlwind(caster, skill, weaponSystem, effectSystem, dx, dy, dist) {
    EffectLib.addAoeMeleeEffect(effectSystem, caster.x, caster.y, ...);
}

// 每帧调用 — 旋转、移动、粒子拖尾等视觉效果
export function executeWhirlwindTick(caster, skill, effectSystem, dt) {
    caster.angle += dt * 18;  // 高速旋转
    // 向目标移动、粒子拖尾...
}

// 每个伤害间隔调用 — 真正造成伤害
export function executeWhirlwindDamage(caster, skill, effectSystem) {
    // 对范围内敌人造成 damage 并吸血
}
```

**2b-2.** 在 `SkillRegistry` 的 `channelHandlers` 中注册：
```js
import { executeWhirlwind, executeWhirlwindTick, executeWhirlwindDamage } from './abilities/Whirlwind.js';

const channelHandlers = {
    whirlwind: {
        tick:   executeWhirlwindTick,    // 每帧
        damage: executeWhirlwindDamage,  // 每间隔
    },
};
```

**2b-3.** 在角色技能数据中指定间隔：
```js
skill: {
    type: 'whirlwind',
    duration: 2.0,               // 总持续时间
    channelTickInterval: 0.25,   // 伤害间隔（秒）
}
```

**Fighter 的 `channeling` 状态是通用的**——它只做三件事：
1. 每帧调用 `executeChannelTick()` → 转发到 handler 的 `tick`
2. 每隔 `channelTickInterval` 秒调用 `executeChannelDamage()` → 转发到 handler 的 `damage`
3. `channelTimer` 归零后自动退出

`skill` 状态中的转场也已通用化——检测 `isChannelingSkill(skill.type)`，不再硬编码具体技能名。

**以后新增持续施法技能**（如激光、旋风、剑刃风暴），只需创建执行器文件 + 在 `channelHandlers` 中加一条，Fighter 不用动。

### 步骤 3：注册角色

在 `js/characters/index.js` 中：

```js
// 1. 添加 import
import { berserker } from './Berserker.js';

// 2. 添加到 characterData 对象
export const characterData = {
    swordsman,
    archer,
    // ...
    berserker,      // ← 新增
    // ...
};
```

> 注册后，角色会自动出现在选择界面。`characterData` 中的顺序决定 UI 中的显示顺序。

### 步骤 4：添加被动效果（如果需要）

被动效果的实现分为两步。

**4a. 在角色数据中声明：**

```js
passives: [
    { id: 'blood_rage',
      name: '血怒',
      description: '生命值越低攻击速度越快。' }
],
```

**4b. 在 Fighter 类中实现逻辑。**

简单被动可以直接在 Fighter 中检查 `hasPassive('id')`。

项目的速度和攻速修改已经 **集中化** 为两个方法，避免分散在多个文件中重复判断：

- `Fighter.getAttackTimerRate()` — 攻速倍率
- `Fighter.getSpeedMultiplier()` — 移速倍率

**这两个方法已被 `Fighter.update()` 和 `FighterAI` 统一调用**，新增被动只需改这里，不需要到处改。

**攻击速度修改** — 在 `js/fighter.js` 的 `getAttackTimerRate()` 方法中添加：

```js
getAttackTimerRate() {
    let rate = this.buffs.slowAttackRate(); // 减速时返回 0.6，否则 1.0
    if (this.hasPassive('blood_rage')) {
        const hpPercent = this.hp / this.maxHp;
        rate *= (1 + (1 - hpPercent) * 1.5);
    }
    return rate;
}
```

> 调用链：`Fighter.update()` 第 169 行 `this.getAttackTimerRate()` → 控制攻击冷却缩减速度。

**移动速度修改** — 在 `getSpeedMultiplier()` 方法中添加：

```js
getSpeedMultiplier() {
    let mult = this.isSlowed() ? 0.5 : 1.0;
    if (this.hasPassive('blood_rage') && this.hp < this.maxHp * 0.35) {
        mult *= 1.25;
    }
    return mult;
}
```

**⚠️ 重要：** 新增移速被动后，必须确保 `FighterAI` 使用的是 `getSpeedMultiplier()` 而不是内联的三元表达式。本项目已做此重构：

- `js/ai/FighterAI.js` 第 70 行（`applyMovement`）：
  ```js
  // 旧代码（分散、不利于扩展）：
  var moveSpeed = baseSpeed * dt * (this.fighter.isSlowed() ? 0.5 : 1.0);
  // 新代码（集中、被动可复用）：
  var moveSpeed = baseSpeed * dt * this.fighter.getSpeedMultiplier();
  ```

- `js/ai/FighterAI.js` 第 247 行（`applyRepositionMovement`）：同上替换。

> 否则新被动不影响位移，因为 `FighterAI` 还在用旧的内联判断。

**伤害/受击修改** — 在 `js/skills/Passives.js` 中添加函数，然后在对应钩子中调用：

| 钩子函数 | 调用时机 | 典型用途 |
|----------|----------|----------|
| `applyMeleeHitPassives()` | 近战命中时 | 溅射、穿透、附魔 |
| `tryDamageAvoidancePassives()` | 受伤前 | 闪避、格挡 |
| `applyDamageReductionPassives()` | 受伤时 | 减伤、护盾 |
| `tryLethalSurvivalPassives()` | 致命伤害时 | 重生、不屈 |

### 步骤 6：绘制角色外观

`drawDecorations(ctx, x, y, angle, size, time)` 是实现角色外观的函数：

```js
drawDecorations: function(ctx, x, y, angle, size, time) {
    ctx.save();
    ctx.translate(x, y);     // 平移到角色中心
    ctx.rotate(angle);       // 旋转到角色朝向（+X = 前方）

    // 在这里绘制武器、角、翅膀等
    // 使用 time 参数做动画

    ctx.restore();
}
```

**重要：**
- 必须先 `save()` 后 `restore()`，避免影响其他渲染
- `+X` 方向是角色正前方（朝向敌人）
- 使用 `size` 做比例计算，适配不同角色尺寸
- 用 `Math.sin(time * N)` 做循环动画

## 完整示例：狂战士

### 特性

- **被动 - 血怒**：血量越低攻速越快（最高 2.5×），HP < 35% 时移速 +25%
- **技能 - 大风车**：持续 2 秒旋转，每 0.25 秒对周围敌人造成伤害并吸血
- **武器**：双斧，带角盔

### 涉及文件

| 文件 | 作用 |
|------|------|

## 常见问题

### Q: 角色没有出现在选择界面？
- 检查 `js/characters/index.js` 是否 import 并加入了 `characterData`
- 确认没有设置 `hidden: true`

### Q: 技能不生效？
- 检查 `skill.type` 是否已在 `SkillRegistry.js` 的 `skillExecutors` 中注册
- 检查技能文件是否被正确 import

### Q: 被动不生效？
- 确认 `passives[].id` 拼写与 `hasPassive('id')` 完全一致
- 确认在 Fighter 中正确调用了 `hasPassive()`

### Q: 外观不显示？
- 确认 `drawDecorations` 是一个 function，且参数名与函数内使用一致
- 检查是否正确使用了 `ctx.save()/restore()` 和 `ctx.translate()/rotate()`
