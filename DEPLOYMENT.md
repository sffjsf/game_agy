# 项目部署与跨设备迁移指南 (Deployment Guide)

这个 2D 自动战斗竞技场项目是一个**纯前端项目**。它完全由纯静态文件组成（`HTML`, `CSS`, `JS`），**不需要**复杂的后端服务器环境（如 MySQL 数据库、Java/Node.js 后端等），也没有通过 Webpack/Vite 等工具进行打包。

这就意味着，**将该项目部署到另一台电脑或者公网服务器上极其简单**。以下是具体的几种做法：

---

## 方法一：离线纯本地运行（最简单的方式）

如果您只是想把游戏发给朋友或者换个电脑本地玩，不需要任何配置：

1. **打包文件**：将整个项目文件夹（包含 `index.html`、`css/` 和 `js/` 文件夹）打包成一个 `.zip` 压缩包。
2. **发送并解压**：通过微信、U盘等发送到另一台电脑上，然后解压缩。
3. **直接运行**：在另一台电脑上，只需**双击打开 `index.html` 文件**。由于浏览器内置了对 HTML 和原生 Web Audio API 的支持，游戏一般可以直接运行！

> **⚠️ 注意**：如果后续您为项目添加了复杂的本地图片或使用 ES Module 模块，浏览器的“跨域同源策略”可能会阻止 `file:///` 协议加载部分资源（导致白屏）。这时候请使用下面的【方法二】。

---

## 方法二：本地小型服务器运行（推荐用于跨设备测试）

当项目由于浏览器安全策略在双击打开时出现问题，或者您希望在局域网内用手机浏览器联机预览，可以使用一条命令开启本地服务。

前提：目标电脑安装了 Python 或 Node.js。

1. **进入目录**：打开目标电脑的命令行终端（Terminal / CMD / PowerShell），使用 `cd` 命令进入到解压后的项目根目录。
2. **启动服务**：
   - **如果对方有 Python (Mac 通常自带)**：
     ```bash
     python3 -m http.server 8080
     ```
   - **如果对方有 Node.js**：
     ```bash
     npx serve -p 8080
     ```
3. **访问游戏**：
   - 在该电脑的浏览器输入：`http://localhost:8080`
   - 如果要让处于同一 Wi-Fi 下的手机或另一台电脑访问，请找到运行服务电脑的局域网 IP（比如 `192.168.1.100`），在手机浏览器输入：`http://192.168.1.100:8080`。

---

## 方法三：免费部署到公网 (Vercel / GitHub Pages)

如果您想把游戏挂在互联网上，让所有人都能通过一个链接玩到，无需花钱购买服务器。这是最推荐的“真·部署”方案：

### 方案 A：使用 Vercel (最快，体验最好)
1. 将这个项目的所有代码推送到您的 **GitHub 仓库**。
2. 登录 [Vercel](https://vercel.com/)，选择使用 GitHub 账号授权登录。
3. 点击 **"Add New..." -> "Project"**，导入您刚刚创建的 GitHub 仓库。
4. 不需要修改任何构建配置（Vercel 会自动识别出这是静态网页），直接点击 **"Deploy"**。
5. 等待 10 秒钟，Vercel 就会为您生成一个全世界都可以访问的游戏公网网址！

### 方案 B：使用 GitHub Pages (适合开源分享)
1. 将代码推送到 GitHub，仓库名可以是任意名称，也可以是特殊的 `<你的用户名>.github.io`。
2. 进入该仓库的网页，点击右上角的 **Settings (设置)**。
3. 在左侧菜单栏找到 **Pages** 选项。
4. 在 **Build and deployment** 的 Source 设置下，选择 `Deploy from a branch`。
5. 在 Branch 下拉框中选择 `main`（或 `master`），然后点击 Save 保存。
6. 等待一两分钟后，GitHub 会提示您的站点已经发布，并提供一个类似 `https://your-username.github.io/game-repo/` 的网址。

## 常见问题排查 (FAQ)

**Q：为什么部署到另一台电脑后，听不到声音了？**
A：浏览器为了防止网页乱弹广告扰民，有**自动播放限制策略**。必须要求用户与网页有交互（例如点击一下任何按钮、选择英雄），浏览器才会允许 JavaScript 的 `AudioContext` 播放声音。只需用鼠标点一下“英雄图鉴”或者“开始按钮”，声音系统就会自动激活。

**Q：有些电脑画面尺寸好像不对齐？**
A：我们的项目使用了原生的 `window.innerWidth` 监听并调整 Canvas 尺寸。如果您发现画面拉伸，尝试缩小一下浏览器窗口再全屏，Canvas 的 `resize` 事件会自动修正画面比例。
